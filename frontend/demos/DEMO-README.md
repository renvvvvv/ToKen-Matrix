# 🎬 Token Matrix Demo 展示效果集合

三个完整的 HTML 演示页面，展示机房智能运维 AI Agent 系统的核心业务流程。所有 Demo 均为**纯前端展示**，使用虚拟数据和动画效果，无需后端服务。


## 🌐 Demo 01: Web Surfer - LangGraph 节点轨迹流动

**文件:** `demo-01-web-surfer.html`

### 功能描述
展示 LangGraph 智能搜索与信息提取的完整流程，从用户查询到推荐结果的端到端执行。

### 布局设计
- **左侧 (1/3):** 流程节点时间线
  - 7 个关键节点: START → Router → Search → Extract → Backtest → Summarize → END
  - 实时脉冲高亮动画显示当前执行节点
  - 节点流动时间 800ms/步，完整流程循环约 6 秒

- **中央 (1/3):** LLM 推理弹窗
  - 显示诊断推理链路的详细步骤
  - 包括问题分析、Router 决策、搜索结果、特征提取、回测结论
  - 橙色边框强调活动状态

- **右侧 (1/3):** 反事实推理结果卡片
  - **推荐行动:** 基于市场环境的投资建议
  - **反事实对比:** "若市场下跌X%，基金预期表现..."
  - **飞书通知:** 通知已下发的状态提示

### 交互点
- **▶ 开始流程** 按钮: 手动触发整个流程
- **⟲ 重置** 按钮: 重置流程状态
- 页面加载后自动启动一轮流程演示

### 动画效果
- 节点脉冲: 橙色发光 + 1.1x 放大
- 节点滑入: 200ms 侧向滑入动画
- 卡片弹入: 0.4s 弹性缩放
- 自动循环: 6s 完整流程 + 1.5s 重置间隔

## 📊 Demo 02: 预测推荐审批执行流程

**文件:** `demo-02-predict-recommend.html`

### 功能描述
展示机房冷却调整的完整决策闭环：预测 → 推荐 → 审批 → 执行。

### 布局设计
- **左侧 (1/3):** 温度预测图表
  - Canvas 绘制的动态曲线
  - 显示当前、5分钟、15分钟、30分钟的温度预测
  - Y 轴温度范围 30-36°C，X 轴时间范围 0-30 分钟
  - 数据点用不同颜色标识 (当前:青、5min:橙、15min:红、30min:橙)
  - 实时监测数据卡片: 当前温度、多时间段预测值

- **中央 (1/3):** 三个候选动作卡片
  - **动作 A (低风险):** 增加风扇转速 10-15% - 效果 85/100
  - **动作 B (中风险):** 关闭非关键服务 - 效果 92/100
  - **动作 C (高风险):** 启动应急冷却系统 - 效果 98/100
  - 卡片包含风险等级徽章、描述、度量指标
  - 动作 A 默认选中（推荐）

- 流程指示器
  - 4 个步骤点: 预测 → 推荐 → 审批 → 执行
  - 前两个默认激活（橙色脉冲）

- **右侧 (1/3):** 人工审批 + 执行反馈
  - **审批卡:** 显示当前风险等级 (MED 中风险)、系统推荐
  - **审批按钮:** [✓ 批准] (绿色) / [✗ 拒绝] (红色)
  - **执行反馈 (初始隐藏):**
    - 指令下发成功
    - Hermes 准则验证 (业务连续性、功耗监控、告警调整)
    - 实时监测中 (预计 2 分钟内降温)
  - **执行完成卡 (延迟显示):** 温度已降至 31.8°C，关闭时间 3 分钟

### 交互点
- **[✓ 批准]** 按钮:
  - 点击后显示执行反馈卡片 (淡入动画)
  - 2 秒后显示执行完成卡片
  - 进度条流畅推进

- **[✗ 拒绝]** 按钮:
  - 弹出提示信息

- Canvas 图表响应式自适应

### 动画效果
- 流程指示器脉冲: 1.5s 循环，带发光阴影
- 卡片选中: 0.3s 过渡，橙色边框 + 背景淡化
- 反馈卡片推入: 0.5s 淡入 + 滑入
- 完成卡片: 0.4s 弹性出现

## 🚨 Demo 03: Hermes 多场景告警触发完整链路

**文件:** `demo-03-hermes-scenarios.html`

### 功能描述
展示告警检测、智能分类、Skill 执行、通知推送的完整业务流程，支持 4 种场景切换。

### 布局设计
- **左侧 (1/3):** 场景选择与触发条件
  - **场景下拉菜单:**
    - 冷却系统故障 (默认)
    - 电源不稳定
    - 火灾预警
    - 网络中断
  
  - **场景描述:** 动态更新，解释当前场景的业务背景
  
  - **触发条件卡片:** 显示 3 条具体的告警触发规则
  
  - **[🔔 触发告警]** 按钮: 启动完整演示流程

- **中央 (1/3):** 执行链路流程图
  - 4 个阶段 (纵向布局):
    1. 🔍 **告警检测** - 规则匹配 → 触发条件验证
    2. ⚙️ **智能分类** - 问题类型识别 + 严重程度评估
    3. ⚡ **Skill 调用** - 执行对应的自动化应对脚本
    4. 📢 **通知推送** - 多渠道告知相关人员
  
  - 每个阶段包含:
    - 圆形图标 (初始青色，激活时橙色脉冲)
    - 阶段标题 + 描述
    - 连接线 (初始淡，激活时橙色渐变)

  - **状态面板 (下方):**
    - 场景状态: "就绪" → "执行中..." → "已完成"
    - 执行进度: 0% → 25% → 50% → 75% → 100%
    - 技能状态: "等待中" → "✓ [对应技能名]" (绿色)
    - 通知计数: 0 → 1 → 2 ...

- **右侧 (1/3):** 通知卡片栈
  - **初始状态:** "👀 触发告警后，通知卡片将显示在此"
  
  - **触发后 (依次弹出):**
    - **飞书通知卡** (青色边框)
      - 标题: 🔴/⚠️/🌐 + 告警类型
      - 徽章: CRITICAL / WARNING / 工单 ID
      - 正文: 告警详情 + 自动应对措施说明
      - 操作按钮: [查看详情] / [确认处理] 等
    
    - **OpenClaw 工单卡** (橙色边框)
      - 标题: 🔨 工单系统 - 自动派单
      - 工单号、级别、类型
      - 负责人与 ETA
      - 操作按钮: [接收] / [转派] 等

  - 卡片堆叠显示 (纵向错开 120px)
  - 每张卡片 0.8s 间隔推入

### 场景详情

#### 1️⃣ 冷却系统故障 (默认)
- **触发条件:**
  - 温度 > 40°C 持续 5 分钟
  - 制冷风扇转速 < 30% 且持续下降
  - 冷却液温度异常上升
- **技能:** 启动应急制冷
- **严重程度:** CRITICAL (红色)
- **通知:** 飞书 + 工单派单

#### 2️⃣ 电源不稳定
- **触发条件:**
  - 输入电压 > 240V 或 < 210V
  - 连续告警 3 次以上
  - 负载不均衡超过 20%
- **技能:** 切换备用电源
- **严重程度:** HIGH (橙色)
- **通知:** 飞书告警

#### 3️⃣ 火灾预警
- **触发条件:**
  - 烟雾浓度 > 1.5 ppm
  - 温度上升速率 > 5°C/min
  - 烟火探测器连续触发
- **技能:** 启动自动灭火系统
- **严重程度:** EMERGENCY (红色脉冲)
- **通知:** 飞书 + 消防应急工单

#### 4️⃣ 网络中断
- **触发条件:**
  - 核心交换机 ping 失败 3 次
  - BGP 路由表异常变化
  - 带宽利用率 > 95%
- **技能:** 切换备用网络路由
- **严重程度:** HIGH (橙色)
- **通知:** 飞书告警

### 交互点
- **场景下拉菜单:** 选择不同场景，更新所有相关信息
- **[🔔 触发告警]** 按钮:
  - 启动 4 阶段流程，每阶段 1.2s
  - 实时更新状态面板
  - 依次弹出通知卡片

### 动画效果
- 阶段激活: 1.2s 过渡，图标脉冲发光
- 连接线流动: 连续动画，渐变效果
- 状态更新: 数字平滑过渡
- 通知卡片: 0.5s 侧向滑入 + 淡入
- 完整演示时间: 4.8s (4 阶段) + 1.6s (2 张卡片) = 约 6.4s

## 🎨 设计系统

所有 Demo 共享统一的设计系统 (`style.css`)：

### 色调 (融合系统主色调)
- **主深蓝:** #0f1419 - 背景基础
- **科技蓝:** #1e3a8a - 卡片背景
- **橙色强调:** #f97316 - 活动状态、按钮
- **青绿强调:** #06b6d4 - 信息、次要元素
- **绿色:** #10b981 - 成功状态
- **错误红:** #ef4444 - 错误、紧急

### 组件库 (shadcn/ui 风格)
- **卡片 (.card):** 玻璃态背景 + 边框 + 阴影
- **按钮 (.btn-*):** 渐变背景、悬停动画
- **徽章 (.badge-*):** 小型状态指示
- **时间线 (.timeline):** 节点流动效果
- **弹窗 (.modal):** 居中弹性出现

### 动画库
- **脉冲:** 1.5s 循环，用于激活状态
- **滑入:** 0.6s 侧向进入
- **弹出:** 0.4s 弹性缩放
- **流动:** 2s 线性连续动画

### 响应式
- 全屏铺满，无滚动条
- 超小屏幕时转为列布局
- Canvas 图表自适应 DPR

## 🚀 使用方法

### 本地预览
直接在浏览器中打开 HTML 文件，无需任何服务器：

```bash
# 方式 1: 直接打开
file:///path/to/demo-01-web-surfer.html

# 方式 2: 使用本地服务器 (建议)
cd token-matrix/frontend/demos/
python -m http.server 8000
# 访问 http://localhost:8000/demo-01-web-surfer.html
```

### 嵌入到网站
在你的主应用中通过 iframe 嵌入：

```html
<iframe 
  src="demos/demo-01-web-surfer.html" 
  style="width: 100%; height: 100vh; border: none;">
</iframe>
```

### 自定义样式
修改 `style.css` 中的 CSS 变量即可全局改色：

```css
:root {
  --accent-orange: #your-color;
  --accent-cyan: #your-color;
  /* ... */
}
```
